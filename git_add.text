### What is `git add`?

`git add` is used to **move changes from the working directory to the staging area**.
It tells Git: *“Include these changes in the next commit.”*

---

## 🔧 Syntax

```bash
git add <file>
```

---

# 🧠 Core Concept

Git has 3 areas:

1. **Working Directory** → where you edit files
2. **Staging Area** → prepared changes
3. **Repository** → committed history

👉 `git add` moves changes from **Working → Staging**

---

## ✅ Example 1: Add a single file

```bash
echo "Hello Git" > file.txt
git status
```

👉 Output:

```bash
Untracked files:
  file.txt
```

Now add it:

```bash
git add file.txt
git status
```

👉 Output:

```bash
Changes to be committed:
  new file: file.txt
```

---

## ✅ Example 2: Add all files

```bash
git add .
```

👉 Adds:

* All new files
* All modified files

---

## ✅ Example 3: Add multiple files

```bash
git add file1.txt file2.txt
```

---

## ✅ Example 4: Add only modified files

```bash
git add -u
```

👉 Adds:

* Modified files
* Deleted files
  ❌ Not new files

---

## ✅ Example 5: Interactive add (advanced)

```bash
git add -p
```

👉 Lets you:

* Add specific lines (partial changes)

---

# 📊 Before vs After `git add`

| State Before | After `git add` |
| ------------ | --------------- |
| Untracked    | Staged          |
| Modified     | Staged          |

---

# 🔁 Full Flow Example (Important)

```bash
git init
echo "A" > file.txt

git status        # untracked
git add file.txt
git status        # staged

git commit -m "Add file"
```

---

# ⚠️ Common Mistakes

❌ Thinking `git add` commits code
👉 It only **prepares** changes

❌ Using `git add .` blindly
👉 May add unwanted files

---

# 🧠 Key Points (Interview Ready)

* `git add` stages changes for commit
* It does NOT save changes permanently
* You can control what goes into a commit
* Supports selective staging (`-p`)

---

# 💡 Pro Tip

Check before commit:

```bash
git status
```

👉 Always verify staged files
