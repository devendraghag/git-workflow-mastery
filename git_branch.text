Here’s a **clear, practical guide to `git branch`** with explanations and real examples you can even use in interviews or your GitHub demo repo.

---

# 🔹 What is `git branch`?

A **branch** in Git is a separate line of development. It allows you to work on features, fixes, or experiments **without affecting the main codebase**.

Think of it like:

> 👉 Main branch = stable code
> 👉 Feature branch = your working copy

---

# 🔹 Basic Syntax

```bash
git branch <branch-name>
```

---

# 🔹 1. List All Branches

### Command:

```bash
git branch
```

### Example:

```bash
git branch
```

### Output:

```
* main
  feature-login
  bugfix-header
```

### Explanation:

* `*` → current active branch
* Others → available branches

---

# 🔹 2. Create a New Branch

### Command:

```bash
git branch feature-payment
```

### Example:

```bash
git branch feature-auth
```

### Explanation:

* Creates a branch from current branch
* Does **NOT switch** to it

---

# 🔹 3. Switch to a Branch

### Command:

```bash
git checkout <branch-name>
```

### Example:

```bash
git checkout feature-auth
```

### Modern alternative:

```bash
git switch feature-auth
```

### Explanation:

* Moves your working directory to that branch

---

# 🔹 4. Create + Switch (Most Used)

### Command:

```bash
git checkout -b <branch-name>
```

### Example:

```bash
git checkout -b feature-dashboard
```

### Modern:

```bash
git switch -c feature-dashboard
```

### Explanation:

* Creates AND switches in one step
* Common in real projects

---

# 🔹 5. Delete a Branch

### Command:

```bash
git branch -d <branch-name>
```

### Example:

```bash
git branch -d feature-auth
```

### Force delete:

```bash
git branch -D feature-auth
```

### Explanation:

* `-d` → safe delete (only if merged)
* `-D` → force delete

---

# 🔹 6. Rename a Branch

### Command:

```bash
git branch -m <new-name>
```

### Example:

```bash
git branch -m feature-login login-feature
```

---

# 🔹 7. View All Branches (Local + Remote)

### Command:

```bash
git branch -a
```

### Output Example:

```
* main
  feature-ui
  remotes/origin/main
  remotes/origin/feature-ui
```

---

# 🔹 8. Track Remote Branch

### Command:

```bash
git branch --track <branch> origin/<branch>
```

### Example:

```bash
git branch --track feature-api origin/feature-api
```

---

# 🔹 Real Workflow Example (Interview-Level)

```bash
# Step 1: Clone repo
git clone https://github.com/user/project.git

# Step 2: Create feature branch
git checkout -b feature-login

# Step 3: Make changes
git add .
git commit -m "Added login feature"

# Step 4: Push branch
git push origin feature-login

# Step 5: Merge later into main
git checkout main
git merge feature-login

# Step 6: Delete branch
git branch -d feature-login
```

---

# 🔹 Pro Tips (Good for Interviews)

* Always create branches for features:

  ```
  feature/*
  bugfix/*
  hotfix/*
  ```
* Keep `main` clean and production-ready
* Use meaningful branch names:
  ❌ `test123`
  ✅ `feature-user-authentication`
