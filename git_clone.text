### What is `git clone`?

`git clone` is used to **copy an existing remote repository** (usually from GitHub) to your local machine.

It not only downloads files, but also:

* Full **commit history**
* All **branches**
* Remote connection (`origin`)

---

## 🔧 Syntax

```bash
git clone <repository-url>
```

---

## ✅ Example 1: Clone a GitHub repository

```bash
git clone https://github.com/user/project.git
```

👉 Output:

```bash
Cloning into 'project'...
remote: Enumerating objects...
Receiving objects: 100%...
```

📌 Result:

* A new folder `project/` is created
* Contains all files + `.git` directory
* Automatically connected to remote (`origin`)

---

## ✅ Example 2: Clone into custom folder name

```bash
git clone https://github.com/user/project.git my-folder
```

📌 Result:

* Repo will be downloaded into `my-folder/`

---

## ✅ Example 3: Clone specific branch

```bash
git clone -b feature-branch https://github.com/user/project.git
```

📌 Only that branch will be checked out after cloning

---

## ✅ Example 4: Shallow clone (faster)

```bash
git clone --depth 1 https://github.com/user/project.git
```

📌 Downloads only latest commit (no full history)

---

## 📂 What happens after clone?

```bash
project/
 ├── .git/        ← full git history
 ├── files...
```

Also automatically set:

```bash
origin → https://github.com/user/project.git
```

Check using:

```bash
git remote -v
```

---

## 🔄 Clone vs Init (Important Interview Question)

| Feature      | git init | git clone              |
| ------------ | -------- | ---------------------- |
| Creates repo | ✅ Yes    | ❌ No (copies existing) |
| Remote code  | ❌ No     | ✅ Yes                  |
| History      | ❌ No     | ✅ Yes                  |
| Setup remote | ❌ Manual | ✅ Automatic            |

---

## 🧠 Key Points (Interview Ready)

* `git clone` copies **entire repository + history**
* Automatically sets remote as `origin`
* Used when working on **existing projects**
* Saves time vs manual setup (`git init + remote add`)

---

## 💡 Pro Tip

After cloning:

```bash
cd project
git status
```

👉 Always verify repo state before starting work
